# Traffic Patterns

The traffic pattern determines the spatial distribution of packets in the network, dictating the destination node for each packet generated by a source node. These patterns are implemented in [traffic.cpp](../../booksim/src/traffic.cpp) and configured via the `traffic` parameter.

---

## 1. Uniform Random (`uniform`)

- **Description**: The destination is chosen uniformly at random from all possible nodes in the network.
- **Characteristics**: This is the most standard baseline traffic pattern. It distributes load evenly across the network, making it excellent for measuring peak bisection bandwidth.

## 2. Permutation Traffic

Permutation patterns map each source node to a specific destination node mathematically based on their bit addresses. They stress specific paths in the network.

### 2.1 Bit-Complement (`bitcomp`)
- **Description**: The destination ID is the bitwise complement of the source ID (i.e., all 1s become 0s and vice versa).
- **Use Case**: Excellent for stressing meshes, as packets must often cross the exact center of the network.

### 2.2 Transpose (`transpose`)
- **Description**: Treats the node address as coordinates `(i, j)` and sends the packet to `(j, i)`.
- **Use Case**: Simulates the communication pattern of matrix transpose operations.

### 2.3 Shuffle (`shuffle`)
- **Description**: The destination ID is a cyclic left shift of the source ID's binary representation.
- **Use Case**: Common in parallel algorithms like the Fast Fourier Transform (FFT).

### 2.4 Tornado (`tornado`)
- **Description**: In an N-dimensional radix-K network, the destination coordinate in each dimension is shifted by $(k/2 - 1)$. 
- **Use Case**: Highly stressful for torus networks, testing the longest possible minimal paths.

## 3. Localized Traffic

### 3.1 Neighbor (`neighbor`)
- **Description**: The destination is the immediate adjacent node (offset by 1) in the first dimension.
- **Use Case**: Tests nearest-neighbor communication and point-to-point latency without stressing the network core.

## 4. Hotspot Traffic (`hotspot`)

- **Description**: Simulates a network where one or more specific nodes (hotspots) receive a disproportionately high amount of traffic, representing servers or memory controllers.
- **Mechanics**: Instead of a purely uniform distribution, packets have an elevated probability of targeting the designated hotspot nodes.

## 5. Trace-Driven / Custom Traffic

While BookSim relies heavily on the synthetic traffic models listed above, it can be extended or modified to ingest trace files containing specific (Source, Destination, Timestamp) tuples generated by full-system simulators.
